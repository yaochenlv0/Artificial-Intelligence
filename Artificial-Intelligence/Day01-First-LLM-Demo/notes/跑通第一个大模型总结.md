# 基本流程
1. 创建调用API的对象（给出认证密钥、调用请求的服务商的服务器）
2. 获取模型的回答结果（指出模型类型，调用模型的API工具，按照列表形式将对话发送给模型，模型根据对话的上下文生成回答，并返回给response）
3. 打印回答（取出response中的choices，从choices中取出某一个choice，从choice中取出message，从message中取出content（回答））
# API是什么
## 形象理解
API就相当于一个中介，A想要获取B的服务，就需要向中介提供请求服务的对象，想干啥，以及干这件事所需要的基本信息，然后就可以调用B的服务了。
## 程序层面
放到程序层面，就是程序A想调用程序B的服务，就需要向API提供端点（B的地址），方法（使用B进行什么样的操作）、参数（进行这个操作所需要的基本信息）
## API的三个核心要素

| 要素      | 说明        | 示例                                  |
| ------- | --------- | ----------------------------------- |
| 端点（url） | API的地址    | https://deepseek.com/v1/chat        |
| 方法      | 要进行什么操作   | GET(获取),POST(提交),PUT(更新),DELETE(删除) |
| 参数      | 传递给API的信息 | {"city": 北京, "date": "2026-04-19"}  |
