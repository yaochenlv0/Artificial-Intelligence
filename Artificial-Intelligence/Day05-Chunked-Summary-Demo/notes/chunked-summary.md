# 基本流程
### 1.创建客户端
1. 大模型接口密钥
2. 大模型地址
### 2.读取文本函数
1. 读取指定的文本，并且读取到的内容，放入text参数，并返回
2. 如果执行过程中出现错误，就打印错误
### 3.切分文本函数
1. 输入切分的文本，以及切分的方式（按照文本字符大小进行切分）
2. 创建chunks列表
3. 根据给出的字符大小，将文本依次切分为指定的字符大小的chunk，并且chunk中的内容依次添加到chunks列表当中，并返回
### 4. 总结单个chunk文本函数
1. 发起调用deepseek模型API接口的请求
2. 将单个chunk的messages列表中的信息按照对话的形式发送给大模型
3. 然后大模型基于上下文信息组织并生成回答，并返回单个chunk的总结
4. 如果在执行过程中出错，就打印出错信息
### 5.对所有chunk分别总结函数
1. 输入chunks列表
2. 创建单个chunk总结列表chunk_summaries
3. 将chunks列表中的每个chunk依次调用总结单个chunk文本函数，并将单个chunk总结结果添加到单个chunk总结列表chunk_summaries中，并返回
### 6.汇总所有分段摘要函数
1. 将文本列表chunk_summaries中的文本元素隔一行的方式连接在一起，并返回给summaries_text
2. 请求调用deepseek的API接口
3. 将messages列表以对话的形式发送给大模型
4. 让大模型基于上下文组织并生成汇总摘要的回答，并返回
5. 如果在执行过程中出现错误，就打印错误
### 7.主函数
1. 指定文本文件地址
2. 调用读取文本文件函数，并将结果返回给text
3. 如果文本文件为空，就返回空
4. 调用切分文本函数，输入参数text以及切分的字符大小，并将结果返回给chunks
5. 调用对所有chunk分别总结函数，输入参数chunks，并将结果返回给chunk_summaries
6. 调用汇总所有分段摘要函数，输入参数chunk_summaries，并生成汇总结果final_summaries
7. 如果汇总结果不为空，就打印汇总结果
# 项目理解
### 问题
原来是根据指定的地址读取文本，然后将读取的文本发送给大模型，让大模型生成总结结果。
### 解决方案
由于文本过长，生成的总结内容并不够精准，比较宽泛，因此在将文本发送大模型之前，先将文本按照指定的方式进行切分，对每个切分的chunk进行总结，然后将单个总结结果进行汇总，再将汇总结果发送给大模型进行总结，生成最终总结结果
